두개의 이상의 버튼을 사용할 때는 "btn-set-area" class로 버튼 요소들을 감싸 줍니다.
버튼이 1개 일때 "btn-set-area col-1" class를 추가 합니다.
버튼이 2개 이상 일때 "btn-set-area col-2" class를 추가 합니다.
버튼이 3개 이상 일때 "btn-set-area col-3" class를 추가 합니다.
버튼이 4개 이상 일때 "btn-set-area col-4" class를 추가 합니다.
버튼이 5개 이상 일때 "btn-set-area col-5" class를 추가 합니다.
각 버튼은 특성에 따라 엘리먼트를 시멘틱하게 사용하고, 버튼 사이즈와 버튼 색상 class를 활용하여 디자인 합니다.
col-num을 설정 안할 경우 Tablet(760px) 이하 디바이스는 2column으로 반응형 처리되어 디자인 됩니다.
ex) class="btn-out-size btn-defult" 버튼사이즈(btn-out-size)와 버튼색상(btn-defult) class를 함께 적용합니다.

테이블 내부 버튼 사이즈는 "btn-in-size" class를 적용합니다. (input, select, buttom 등의 요소와 사용)
테이블 외부 버튼 사이즈는 "btn-out-size" class를 적용합니다.
테이블 상세 버튼 사이즈는 "btn-table-view-size" class를 적용합니다.
테이블 게시판 버튼 사이즈는 "btn-table-board-size" class를 적용합니다.

<div class="btn-set-area col-3">
    <label class="btn-out-size btn-blue">
        <span class="fa fa-search before" aria-hidden="true"></span>아이콘 버튼
        <input type="button">
    </label>
    <a href="#" class="btn-out-size btn-gray" role="button">
        아이콘 버튼<span class="fa fa-search after" aria-hidden="true"></span>
    </a>
    <button class="btn-out-size btn-blue" type="button">
        <span class="fa fa-search before" aria-hidden="true"></span>아이콘 버튼
    </button>
</div>
버튼 엘리먼트는 역할에 따라 요소를 사용하고 버튼 사이즈과 버튼 색상 class를 적절히 사용하여 디자인 하시면 됩니다.
아이콘 추가 시 버튼 요소 내부에 <span> or <img> 요소를 사용하여 아이콘(img or FontAwesome) 추가한다.
아이콘을 글자 앞쪽에 배치할 때는 글자 앞으로 아이콘 요소를 배치하고 아이콘 요소에 "before" class를 적용하면 아이콘 정렬이 됩니다.
아이콘을 글자 뒤쪽에 배치할 때는 글자 뒤로 아이콘 요소를 배치하고 아이콘 요소에 "after" class를 적용하면 아이콘 정렬이 됩니다.

<caption> 의 설명은 "(제목)의 타이틀은 (가로 or 세로)정렬로 되어있습니다." 와 같이 일관된 구문을 넣어 사용합니다.
청각만으로도 표의 구조가 그려질 수 있도록 자세히 설명하여 표를 읽어나가는데 도움이 되도록 작성합니다.
각 제목(컬럼)의 너비는 <col> 요소에 width 요소로 너비를 설정하고 너비의 비율에 대비하기 위해 단위를 작성하지 않습니다.
그리고 유동적인 테이블일 경우 컬럼 하나의 값은 width="auto"로 설정하여 변화되는 너비의 값을 대응해야 합니다.
표 마다 일관된 width 값이 없으므로 각 표의 제목(컬럼)의 너비에 따라 재정의하여 사용합니다.
반응형 중단점에 따라 리스트(list-type) 테이블로 변화를 적용할 경우 "table-view-type" class의 형재요소로 "list-type" class를 추가해 줍니다.
ex) <table class="table-view-type list-type">
"list-type"을 빼면 기본 테이블로 적용되고 반응형 적용 시 중단점에 따라 column을 감추는 형태로 사용하면 됩니다.
"list-type"을 사용하는 표의 테이블 제목(컬럼)이 달라질 경우 "table-view-type" class명에 "table-view-type-num" class명과 같이 -num을 붙여서 class명을 정의하고 제목을 css의 content로 재설정 해줘야 합니다.
ex) <table class="table-view-type-01 list-type">

Popup Button

팝업 열기 버튼을 클릭하면 팝업을 호출 합니다. 
<a class="btn-popup-name" role="button" href>팝업 버튼 클릭</a>

팝업버튼의 class="btn-popup-name" 클래스명 클릭 시 해당 팝업의 data-id="popup-name" 와 연결되어 팝업이 띄워지도록 설정 되었습니다.
클래스의 popup-name 은 파일 네이밍과 동일하게 규칙적으로 사용합니다.
